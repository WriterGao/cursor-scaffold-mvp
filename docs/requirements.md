# 需求说明：简单待办列表

## 功能列表

- 用户可以在页面上**新增待办事项**，输入内容后点击按钮添加到列表顶部或底部展示。
- 用户可以在列表中**勾选/取消勾选完成状态**，已完成的待办在样式上与未完成区分（如删除线或颜色变化）。
- 用户可以**删除待办事项**，从当前列表中移除。
- 页面加载时展示一个**空列表**提示，指导用户添加第一条待办。
- 不需要后端持久化，刷新页面后数据可以丢失。

## 用户角色

- **普通访客用户**：打开页面即可使用，无需登录和账号系统。

## 验收标准

- 打开首页 `/` 时可以看到一个标题为「待办列表」或类似文案的页面。
- 在输入框中输入任意文本并点击添加按钮后，该文本会立即出现在列表中。
- 列表项左侧或右侧有勾选控件（如复选框或按钮），点击后该项的样式变为“已完成”，再次点击可恢复为“未完成”。
- 每条待办项旁边有删除按钮，点击后该条待办会从列表中消失。
- 所有操作（添加、勾选、删除）均在前端即时生效，无需页面刷新。
- 页面在桌面端和移动端都有合理布局（不要求复杂响应式，仅需基础适配）。

## 边界条件与不做内容

- 数据仅保存在浏览器内存中，**刷新页面后待办数据丢失是允许的**。
- 不实现用户登录、注册、权限控制。
- 不实现服务端存储或数据库集成。
- 不要求 SEO 复杂优化，只需使用 Next.js 标准页面结构。
- 不实现复杂筛选（如按完成状态过滤）和排序功能，后续如有需要可在新需求中扩展。
